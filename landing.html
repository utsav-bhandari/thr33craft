THR33CRAFT: Project Showcase

This page summarizes what I built for THR33CRAFT, a browser-based voxel sandbox made with Three.js and TypeScript.

Feature 1: World Generation and Exploration

The project streams chunked terrain around the player so the world feels large and continuous while moving in first person.

Feature 2: Block Targeting and Interaction

I implemented block targeting, breaking, and placement workflows to make the sandbox interactive.

Feature 3: Inventory and Hotbar

The inventory supports browsing and selecting blocks, while the hotbar gives fast access during gameplay.

What This Project Demonstrates

THR33CRAFT demonstrates how a real-time 3D sandbox can stay responsive by streaming the world in chunks around the player, then building those chunks nearest-first so nearby terrain appears first while distant terrain catches up in the background.

A big performance detail is voxel culling during meshing. When a block is completely surrounded by solid neighbors on all six sides, it is skipped and never sent to the GPU. Even chunk borders are handled carefully: if a neighboring chunk has not loaded yet, it is treated as solid for meshing so border faces stay hidden instead of popping in and out.

The inventory icons are also generated in an interesting way. Instead of hand-drawing every icon, the project can render each block through a tiny Three.js scene with an orthographic camera and directional light, then pack all results into one texture sheet. Metadata stores each icon's background-position so the inventory and hotbar can display hundreds of block previews from a single image efficiently.

The result is a project that is fun to explore on the surface, but also full of engine-level ideas under the hood: streaming, culling, batching, and UI texture pipelines.
